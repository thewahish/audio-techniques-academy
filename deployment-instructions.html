Phase 1: DigitalOcean Droplet Setup

1.1 Create Droplet

Log into DigitalOcean dashboard
Click "Create" ‚Üí "Droplets"
Choose Ubuntu 22.04 LTS
Select Basic plan ($6/month minimum recommended)
Choose datacenter region closest to your audience
Add SSH key (or use password authentication)
Name your droplet: "audio-academy"
Click "Create Droplet"
Wait for droplet to be ready (2-3 minutes)
Copy the droplet's IP address

Phase 2: Server Configuration

2.1 Connect to Server

Open terminal/command prompt
Connect via SSH:
ssh root@YOUR_DROPLET_IP
Accept the connection (type "yes")
Enter password if prompted

2.2 Install Web Server

Update system packages:
apt update && apt upgrade -y
Install Nginx:
apt install nginx -y
Start and enable Nginx:
systemctl start nginx && systemctl enable nginx
Test: Open browser, visit http://YOUR_DROPLET_IP
Verify you see "Welcome to nginx!" page

2.3 Configure Firewall

Enable UFW firewall:
ufw enable
Allow SSH:
ufw allow ssh
Allow HTTP:
ufw allow 'Nginx Full'
Check status:
ufw status

Phase 3: SSL Certificate Setup

3.1 Install Certbot

Install Certbot:
apt install certbot python3-certbot-nginx -y
Verify installation:
certbot --version

Phase 5: File Upload Preparation

5.1 Prepare Local Files

Open file explorer to audio-techniques-academy folder
Create a ZIP file of the entire folder
Name it: audio-academy.zip
Verify ZIP contains all HTML files and assets

5.2 Upload Files via SCP

Open new terminal/command prompt
Navigate to folder containing audio-academy.zip
Upload ZIP to server:
scp audio-academy.zip root@YOUR_DROPLET_IP:/tmp/
Enter password when prompted
Wait for upload to complete

Phase 6: Server File Setup

6.1 Extract and Deploy Files

Return to SSH session with server
Install unzip:
apt install unzip -y
Navigate to web directory:
cd /var/www/html
Remove default nginx page:
rm index.nginx-debian.html
Extract your files:
unzip /tmp/audio-academy.zip -d .
Move files from subfolder to root:
mv audio-techniques-academy/* . && rmdir audio-techniques-academy
Set proper permissions:
chown -R www-data:www-data /var/www/html && chmod -R 755 /var/www/html

Phase 7: Nginx Configuration

7.1 Configure Virtual Host

Create site configuration:
nano /etc/nginx/sites-available/audio-academy
Paste this configuration (replace YOURDOMAIN.com):
server { listen 80; server_name YOURDOMAIN.com www.YOURDOMAIN.com; root /var/www/html; index index.html unified-audio-academy.html; location / { try_files $uri $uri/ =404; } location ~* \.(css|js|jpg|jpeg|png|gif|ico|svg)$ { expires 1y; add_header Cache-Control "public, immutable"; } }
Save file (Ctrl+X, Y, Enter)
Enable site:
ln -s /etc/nginx/sites-available/audio-academy /etc/nginx/sites-enabled/
Test Nginx configuration:
nginx -t
Reload Nginx:
systemctl reload nginx

Phase 8: SSL Certificate Installation

8.1 Generate SSL Certificate

Run Certbot (replace YOURDOMAIN.com):
certbot --nginx -d YOURDOMAIN.com -d www.YOURDOMAIN.com
Enter email address for notifications
Agree to Terms of Service (A)
Choose whether to share email (Y/N)
Select option 2 (redirect HTTP to HTTPS)
Wait for certificate generation

Phase 9: Final Testing

9.1 Verify Deployment

Open browser
Visit https://YOURDOMAIN.com
Verify site loads with green lock icon
Test main pages work correctly
Check that images load properly
Test both Arabic and English versions
Verify mobile responsiveness

9.2 Setup Auto-Renewal

Test certificate renewal:
certbot renew --dry-run
Verify "Congratulations" message appears
Auto-renewal is now configured

Phase 10: Maintenance Setup

10.1 Create Update Script

Create update script:
nano /root/update-site.sh
Add content:
#!/bin/bash cd /var/www/html # Backup current site tar -czf /root/backups/site-$(date +%Y%m%d-%H%M%S).tar.gz . # Your update commands here systemctl reload nginx
Make executable:
chmod +x /root/update-site.sh
Create backups directory:
mkdir -p /root/backups
